หลังบ้านที่รวมการนำเข้า แก้ไขสถานี และดูคิวรายงานไว้ใน flow เดียว

หน้านี้ออกแบบให้ตัดสินใจเร็วขึ้น: เห็นสถานะระบบด้านบน, กระโดดไปยัง import ได้ทันที, เลือกคิวงานจาก rail ด้านซ้าย, แล้วแก้ข้อมูลจริงใน editor ที่แยกเป็นหมวดชัดเจนโดยไม่ต้องไล่หาช่องกรอกยาวทั้งหน้า

เปิด Import Console จัดการ Stations ตรวจ Reports
01. Snapshot

ดู health ก่อนแก้ข้อมูล

เช็ก auth, write access, Firebase health และ usage ให้ครบก่อนเริ่มแก้หรือนำเข้าข้อมูลรอบใหม่

02. Ingest

ดึงข้อมูลได้ทั้งตรงและจากไฟล์

รองรับทั้งการดึงผ่าน proxy, โหลดไฟล์ JSON จากเครื่อง, หรือวาง payload ตรงเพื่อทดสอบการนำเข้า

03. Triage

คิวงานอยู่ข้างซ้ายทั้งหมด

ค้นหาชื่อปั๊ม แบรนด์ พื้นที่ หรือ ID แล้วเลือก record ที่ต้องแก้โดยไม่หลุดออกจากหน้าหลัก

04. Commit

แก้เป็นหมวดและบันทึกจบในแผงเดียว

editor แยกข้อมูลพื้นฐาน พิกัด รูปภาพ และสถานะน้ำมันเพื่อให้ review ก่อน save ได้ง่ายขึ้น

Ops Status

สถานะระบบสำหรับรอบแก้ไขนี้

มุมนี้ไว้ดูความพร้อมก่อนลงมือจริง ทั้งสิทธิ์การเขียน, health ของ Firebase, และค่าใช้งาน backend

usage ตรงนี้ใช้ดูแนวโน้มค่าใช้จ่ายหลังรัน import หรือใช้งาน backend หนัก ๆ

Admin Access

เข้าสู่ระบบด้วย Google ก่อนเข้า admin

หน้านี้ใช้แก้ข้อมูลสถานีและรายงานบน Firestore โดยตรง จึงเปิดเฉพาะบัญชีที่ผ่านการยืนยันตัวตนและมีสิทธิ์ใช้งานหลังบ้าน

ก่อนเข้าหน้านี้ควรพร้อม
  • Firebase config ถูกต้องและโหลดได้จริง
  • บัญชี Google อยู่ใน allowlist ของ admin
  • กฎ Firestore อนุญาตให้เขียน collection ที่เกี่ยวข้อง
Mission Board

Workflow ของทีมหลังบ้าน

เริ่มจาก snapshot สถานะ, เลือกวิธี ingest, แล้วเข้าสู่คิวงานของสถานีหรือรายงานโดยไม่ต้องสลับหน้าหรือเปลี่ยนบริบท

Import Console Station Workbench Report Workbench
Access

ยืนยันสิทธิ์ก่อนทุกครั้ง

ถ้า write access หรือ Firebase health ยังไม่พร้อม ให้หยุดที่จุดนี้ก่อนเพื่อลดการแก้ข้อมูลไม่สำเร็จ

Import

เลือก source ให้เหมาะกับงาน

มือถือใช้ fetch ผ่าน backend ได้เลย ส่วนงานตรวจ payload ลึก ๆ เหมาะกับไฟล์หรือ textarea มากกว่า

Stations

แก้ข้อมูล canonical ของสถานี

ใช้สำหรับปรับชื่อปั๊ม แบรนด์ พื้นที่ พิกัด รูปภาพ และสถานะน้ำมันหลักที่ public page จะอ้างอิง

Reports

คัดกรองข้อมูลที่ผู้ใช้ส่งเข้ามา

เลือกแก้รายการผิด เปลี่ยนสถานะ ย้าย stationId หรือเก็บ note เพิ่มเติมก่อน sync กลับระบบ

Import Console

นำเข้าข้อมูลจาก PumpRadar แบบมี context ครบก่อนกด submit

รองรับทั้งการดึงข้อมูลสด, import จากไฟล์, และวาง JSON ตรงโดยใช้ `station.id` เป็น Document ID ใน `stations`

Input Modes

เลือกวิธี ingest ตามสภาพแวดล้อมที่ใช้งาน

Mobile

ดึงแล้วนำเข้า

เหมาะกับการทำงานบนมือถือหรือเครื่องที่ไม่มีไฟล์ JSON เตรียมไว้ล่วงหน้า

File

ตรวจ payload ก่อน import

เหมาะกับการ review ไฟล์หลายจังหวัดหรือ payload ที่ export มาจากสคริปต์ของโปรเจกต์

Textarea

วางข้อมูลเพื่อทดลองทันที

ใช้กับการ debug, compare payload หรือแก้ JSON จุดเล็กก่อนอัปโหลดจริง

Checklist ก่อนกด Import
  • เลือกจังหวัดให้ตรงกับ payload ที่จะใช้
  • ถ้าเปิดจาก localhost หรือโดเมนอื่น ให้ใส่ proxy URL ที่ deploy แล้ว
  • ตรวจให้แน่ใจว่า source นี้คือรอบข้อมูลล่าสุดที่ต้องการใช้จริง
ผลลัพธ์ที่ระบบจะสร้างให้
  • สร้าง `fuelStates` มาตรฐานสำหรับหน้า public
  • เติม `reportCount`, `updatedAt` และ metadata ที่จำเป็น
  • เขียนข้อมูลลง Firestore โดยอิง `station.id` ของต้นทาง
วิธีดึง JSON
ถ้า backend function ยังไม่พร้อม ปุ่ม fetch จะใช้ไม่ได้และควรสลับไปโหลดไฟล์ JSON เองแทน
กำลังตรวจสอบ source ของสถานี
Station Queue

เลือกสถานีที่ต้องแก้ไข

ค้นหาจากชื่อ แบรนด์ พื้นที่ หรือ Document ID แล้วเลือก record ที่ต้องทำงานจากคิวนี้

Queue Logic

รายการจะอัปเดตตามคำค้นทันที และเมื่อสร้างสถานีใหม่ editor ด้านขวาจะเปลี่ยนไปเป็น draft สำหรับบันทึก record ใหม่

Live Queue เลือกจากคิวเพื่อโหลดข้อมูลเข้าสู่ editor
Station Editor

เลือกสถานีเพื่อแก้ไข

เลือกสถานีจากคิวด้านซ้าย หรือกดสร้างสถานีใหม่เพื่อเริ่ม draft ใหม่

Canonical Station Data

แผงนี้ใช้แก้ข้อมูลหลักของสถานีที่จะถูกนำไปใช้ต่อในแผนที่ ฟีด และการสรุปรายงาน จัดช่องกรอกเป็น 3 หมวดเพื่อลดการไล่หาฟิลด์

Identity

ข้อมูลอ้างอิงของสถานี

ส่วนนี้กำหนดตัวตนของ record ที่จะถูกเรียกใช้งานจากหน้าสาธารณะ

Geo + Media

พิกัด รูปภาพ และจำนวนรายงาน

ใช้สำหรับเก็บข้อมูลที่ช่วยแสดงผลบนแผนที่และประกอบการตัดสินใจของผู้ใช้

Fuel Matrix

สถานะน้ำมันแต่ละชนิด

ปรับภาพรวมความพร้อมของแต่ละชนิดเชื้อเพลิงในคราวเดียวก่อนบันทึก

สถานะเชื้อเพลิง
Report Queue

จัดการรายงานที่ผู้ใช้ส่งเข้ามา

ใช้สำหรับคัดกรองรายงานผิดพลาด เปลี่ยนสถานะ ย้าย stationId หรือเติมข้อมูลก่อน sync กลับสถานี

Review Queue

รายงานแต่ละรายการจะมีสถานะเชื้อเพลิงและอายุข้อมูลให้ดูทันที ช่วยคัดงานเร่งด่วนได้จากคิวนี้เลย

Review Feed เลือกจากรายการเพื่อเปิด report editor
Report Editor

เลือกรายงานเพื่อแก้ไข

เลือกจากคิวด้านซ้าย แล้วแก้ข้อมูลให้เรียบร้อยก่อนบันทึกกลับ Firestore

User Submitted Data

แผงนี้ใช้ตรวจข้อมูลที่ผู้ใช้ส่งเข้ามา ทั้งสถานี ข้อมูลผู้รายงาน สถานะน้ำมัน พิกัด รูปภาพ และ note ภายในทีม

Reference

ตัวระบุของรายงานและสถานี

ส่วนนี้ใช้ผูก report กับสถานีที่ควรถูก sync กลับไปหลังบันทึก

Status Payload

สถานะน้ำมัน พิกัด และรูปภาพ

ใช้คัดกรอง payload ที่มีผลต่อหน้า public และการรวมข้อมูลของสถานี
